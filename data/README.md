# Data

No data files are included in the project scaffold.

Future data work should document:

- source, access date, license, and confidentiality restrictions;
- raw-to-processed transformations;
- variable definitions and units;
- geographic, sectoral, firm, and time coverage;
- the mapping from each calibration moment to a reproducible input.

Large, licensed, confidential, raw, and processed data are ignored by Git. Keep
only documentation, schemas, small public examples, and reproducible acquisition
or transformation code under version control.
